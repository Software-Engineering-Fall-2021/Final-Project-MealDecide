Application Overview

Meal Decide is supposed to be the solution to the age old question "What do you want to eat?" In this situation, it is meant to be a personal concierge for someone who does not know what to eat or where to eat. This application will allow the user to take a survey/quiz in order to tell the application about meal context and personal taste. First the user will select whether they want to dine in or go out. Depending on the answer, the application will suggest a recipe they can cook (dine in) or a restaurant (go out).

If the user is looking to go out to a restaurant, they will enter their zip code, any dietary accomodations, price range, and answer a set of questions pertaining to taste. Along with that, the user will be able to choose dinner context about wanting fast food, chains restraunts, cheap, sit down, carry out, etc. This is to make sure that the recommendations are appropriate for the occasion. After all, there's a big difference between restaurants you should go to for a formal occasion versus one that's just for a quick coffee run. After taking in the information, the application will then attempt to return 3-5 restraunts in their area that fit within their criteria. If the user still cannot decide, there will be a small spin-the-wheel feature to get a random result based on the set of recommendations.

If the user is looking to dine in, the application will use a similar process to find a recipe recommendation to provide. They will be provided a survey about taste preference, meal type (breakfast, lunch, dinner, or snack), dietary restrictions, number of people, and a similar set of questions pertaining to taste. These questions will attempt to account for what kind of kitchen the user has and how much time they are willing to spend. In the end, a set of recipes will be provided as suggestions - and there will also be a small spin-the-wheel feature to get a random final suggested result if they cannot decide.
